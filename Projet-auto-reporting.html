contexte du projet
Problématique métier
🎯
Objectif
Automatiser entièrement le reporting hebdomadaire qui prenait 3 heures de travail manuel chaque lundi : collecte, consolidation et envoi du rapport.
AutomatisationReporting
❗
Problème
Chaque lundi, un analyste passait 3h à collecter les données de 4 sources différentes, les consolider dans Excel et envoyer manuellement les rapports.
ManuelTemps perdu
💡
Solution
Flux Power Automate déclenché chaque lundi à 7h00 : récupère les données SharePoint, met à jour le fichier Excel maître, génère le PDF et l'envoie automatiquement.
Trigger planifié0 intervention
📈
Impact
3 heures économisées chaque semaine (156h/an). Rapport disponible en boîte mail à 7h15 au lieu de 11h. Zéro erreur de saisie manuelle.
156h économiséesZéro erreur
méthodologie
Comment j'ai construit ce projet
01
Trigger planifié
Configuration du déclencheur récurrent : chaque lundi à 7h00. Paramétrage du fuseau horaire, gestion des jours fériés.
// Trigger : Récurrence Frequence: Semaine Jour: Lundi Heure: 07:00 (Europe/Paris) Activé: Oui
02
Collecte des données
Actions "Get items" pour récupérer les données depuis SharePoint. Filtres dynamiques sur la semaine précédente via expressions formatDate.
// Expression date semaine N-1 formatDateTime( addDays(utcNow(), -7), 'yyyy-MM-dd')
03
Mise à jour Excel
Action "Update row" pour enrichir le fichier Excel maître. Calcul automatique des KPIs via formules pré-définies dans le template.
04
Génération & envoi
Conversion du fichier en PDF via OneDrive. Envoi automatique via Outlook avec corps de mail dynamique incluant les KPIs de la semaine.
// Corps email dynamique "Bonjour, Rapport semaine @{variables('semaine')} : - CA : @{variables('ca_total')}€ - Commandes : @{variables('nb_cmd')} Cordialement, Système automatisé"
